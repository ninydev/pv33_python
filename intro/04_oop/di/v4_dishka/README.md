# Dishka Dependency Injection

`dishka` — це сучасна, швидка та гнучка DI-бібліотека для Python. Вона використовує концепцію "провайдерів" (Providers) та "схоупів" (Scopes - життєвий цикл залежностей).

## Встановлення модулів

Щоб встановити бібліотеку, виконайте команду:
```bash
pip install dishka
```

## Як це працює

1. **Provider**: Ми створюємо клас-наслідник від `Provider` (в нашому випадку `AppProvider`), у якому описуємо фабричні методи для створення об'єктів.
2. **Анотації та Декоратори**:
   - Кожен метод декорується `@provide()`, де обов'язково вказується `scope`. Наприклад, `Scope.APP` створює один екземпляр об'єкта на весь додаток (як Singleton), а `Scope.REQUEST` — новий екземпляр на кожен запит.
   - Типи, що повертаються з фабричних методів (type hints: `-> StorageInterface`), повідомляють Dishka, яку саме залежність вони створюють.
   - Аргументи методу (наприклад, `storage: StorageInterface`) — це ті залежності, які Dishka має автоматично підставити під час створення об'єкту.
3. **Container**: Ми створюємо контейнер за допомогою функції `make_container(provider)`.
4. **Отримання (Get)**: `container.get(AvatarService)` дістає залежність, автоматично проходячи по всьому графу провайдерів. Наприкінці рекомендується робити `container.close()` для правильного завершення роботи.

## Запуск

Просто запустіть файл:
```bash
python main.py
```
